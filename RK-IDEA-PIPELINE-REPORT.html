Executive Summary

The Idea Pipeline is a new feature on the RK Logistics CEO Dashboard that transforms spoken ideas into structured, AI-refined feature requests that flow through an approval process into a developer bucket where implementation prompts are auto-generated for Cursor AI.

API Endpoints

POST /api/transcribe

Accepts audio blob from browser MediaRecorder, sends to OpenAI Whisper (whisper-1 model), returns text transcript. Includes RK-specific vocabulary prompt for accurate transcription of company terms (FTZ, EBITDA, OTT, facility names, etc.).

POST /api/ideas/generate

Takes raw transcript text, uses GPT-4o-mini with a comprehensive RK Logistics system prompt to transform it into a structured idea with: title, summary, problem statement, solution description, business impact, priority level, department assignment, and 3-6 specific functional requirements.

POST /api/ideas/cursor-prompt

Takes an approved, structured idea and generates a comprehensive Cursor AI implementation prompt. The system prompt embeds full platform context: tech stack (React + Express + Tailwind + shadcn/ui), project structure, design system tokens, and critical rules (never break existing functionality, maintain dark mode, etc.).

CRUD: /api/ideas

POST /api/ideas — Create idea. GET /api/ideas — List all (sorted by date). GET /api/ideas/:id — Get one. PATCH /api/ideas/:id — Update status, notes, feedback, cursor prompt, refined data. DELETE /api/ideas/:id — Remove.

Send-Back Feedback Flow

The developer can send any idea back to its original submitter with clarifying questions or feedback. This creates a collaborative loop between leadership and engineering.

1
Developer clicks "Send Back to [Name]" A feedback textarea opens below the detail panel
2
Developer types feedback E.g., "Can you clarify which HubSpot fields contain the license data? Also, should the 90/60/30 day alerts go to the practitioner or to the account manager?"
3
Clicks "Send Feedback" (disabled if empty) Idea status flips to needs_feedback, feedbackNote is saved, any existing cursorPrompt is cleared
4
Idea moves from Dev Bucket back to Pipeline The original submitter sees an amber banner: "Feedback from Developer -- action required" with the note text
5
Submitter addresses feedback and clicks "Resubmit for Review" Idea returns to review status and enters the cycle again

How to Use the Idea Pipeline

Step 1: Record Your Idea

Navigate to Strategy > Idea Pipeline in the sidebar. Click the large teal microphone button. Speak naturally, but make sure to include:

  1. Department name: "For the Sales team..." or "Operations needs..."
  2. Feature request: "I want to add a pipeline visualization that shows..."
  3. Data source: "...pulling data from Insightly deals and QuickBooks invoices..."
  4. Why it matters: "...so reps can see their conversion funnel without switching apps."

Example Recordings

Example 1: "For Sales and BD, I want a deal pipeline visualization that shows stage-by-stage conversion rates. The data should come from Insightly CRM deal stages and close dates. This would help Peter's team identify where deals are getting stuck."

Example 2: "Operations needs a real-time pick-pack dashboard. Data should pull from the warehouse management module showing orders awaiting pick, currently in packing, and shipped today. Include carrier breakdown -- UPS, FedEx, USPS."

Example 3: "For Compliance, we need automated license expiration alerts. Pull practitioner license data from customer records and send email reminders 90 days, 60 days, and 30 days before expiry."

Example 4: "Accounting needs an AR aging dashboard on the platform. Pull aging data from QuickBooks and show 30/60/90/120 day buckets with customer names. David's team is manually pulling this report every week."

Step 2: Review AI Refinement

After recording, the idea appears in the Pipeline list with status "In Review." Click on it to see the AI-structured version with title, summary, problem statement, proposed solution, business impact, priority, department, and specific requirements.

Step 3: Approve to Dev Bucket

If the AI refinement looks good, click "Approve & Send to Dev Bucket". The idea moves to the Dev Bucket tab, where the developer can pick it up.

Step 4: Developer Generates Cursor Prompt

The developer switches to the Developer role toggle, opens the Dev Bucket, selects the idea, and clicks "Generate Cursor Prompt". The AI creates a production-ready implementation prompt with full platform context that can be copied directly into Cursor.

Technical Implementation Details

AI System Prompts

Whisper Transcription Prompt

Includes RK-specific vocabulary: company names (RK Logistics, On Time Trucking, Go Freight Hub), facility names (Patterson, Fremont, Goodyear, Vista Ridge, Kato), business terms (FTZ, EBITDA, BD), systems (Insightly, ZoomInfo), and all 11 department names for maximum transcription accuracy.

GPT Refinement Prompt

Full RK Logistics context: company description (3PL, 12+ facilities, 1.78M sqft), business units (RK Logistics, OTT, Go Freight Hub), customer list (LAM Research, Tesla, Applied Materials, Corning, Netflix), departments, leadership team, and technology stack. Returns structured JSON with 8 fields.

Cursor Prompt Generation

Embeds full platform technical context: React 18 + Vite + Express 5 stack, Tailwind CSS 3 + shadcn/ui design system, project directory structure, CSS custom properties (--primary: 174 72% 33%), Satoshi font, dark mode support, and critical implementation rules.

Persistence Note

Ideas are currently stored in server memory (Map). A restart clears all ideas. For production persistence, the next step is to add a database model (PostgreSQL via Drizzle ORM) with the full Idea schema. The API endpoints are already structured for easy migration to database-backed storage.
